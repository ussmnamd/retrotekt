# Portfolio Rebuild — Master Plan

This document is the **single source of truth** for the portfolio redesign. All agents working on any phase must read this end-to-end before making changes. Update this doc as decisions are made, do not branch into ad-hoc plans.

> **Hard rules — never violate**
>
> 1. **GSAP: one master timeline per page, period.** No `gsap.timeline()` calls scattered across components. Centralize in one parent client component using `useGSAP`. ScrollTrigger animations attach to the master timeline via the `scrollTrigger` config option, not as independent timelines.
> 2. **Don't break existing pages.** GSAP work is scoped to `/portfolio` and `/portfolio/[slug]` only. The homepage Three.js hero and existing animations on `/services`, `/consulting`, `/contact` are off-limits.
> 3. **Dark theme retained** for all portfolio routes (`#0A0A0A` bg, `#F7F0E3` text, `#C4A882` accent). Font: JetBrains Mono (`font-body` / `font-heading`).
> 4. **Mobile-first responsive.** ScrollTrigger pinning and heavy animations gated behind `gsap.matchMedia()` — reduced/no animation on `prefers-reduced-motion`, simplified layout under 768px.
> 5. **Performance budget:** Lighthouse Performance ≥ 85 on the new portfolio page. LCP < 2.5s on cable. CLS < 0.05. Total page weight (excluding video) < 2MB.

## Asset organization (post-pipeline)

Source files live in `portfolio/` (gitignored, source media). Optimized output goes to `public/portfolio/` (committed). Manifest in `app/portfolio/assets.ts` (typed, machine-generated).

```
public/portfolio/
  modesto/
    before/      [01.avif, 01.webp, 01.jpg]   ×3 photos, 3 formats each
    construction/[01.avif, 01.webp, 01.jpg]   ×4 photos
    renders/     [01.avif, 01.webp, 01.jpg]   ×5 renders, 2400/1600/800 widths
    videos/
      walkthrough-01.mp4   (H.264, 1080p, ~10MB)
      walkthrough-01.webm  (VP9 or AV1, ~8MB)
      walkthrough-01.poster.jpg
      hero-loop.mp4        (6s muted clip, ~3MB, autoplay)
      hero-loop.webm
      hero-loop.poster.jpg
  livermore/
    construction/, renders/   (no before, no videos)
  sacramento/
    construction/, renders/
```

**`app/portfolio/assets.ts`** — typed manifest auto-generated by `scripts/build-portfolio-assets.ts`. Pages import from this manifest, never from string paths.

```ts

};
export type ProjectVideo = {
  mp4: string; webm: string; poster: string;
  width: number; height: number;
};
export type ProjectAssets = {
  before: ResponsiveImage[];
  construction: ResponsiveImage[];
  renders: ResponsiveImage[];
  videos: ProjectVideo[];
  heroLoop?: ProjectVideo;
};
export const portfolioAssets: Record<'modesto' | 'livermore' | 'sacramento', ProjectAssets>;
```

## Phase 1 — Asset pipeline (RUNS FIRST, blocks everything)

**Owner:** general-purpose sonnet agent

### 1.1 Reorganize source folders
- Rename source folders to lowercase ASCII slugs in place (or copy):
  - `portfolio/modesto construct beofre/` → working slug `modesto/before/`
  - `portfolio/modesto construct/` → `modesto/construction/`
  - `portfolio/modesto renders/` → `modesto/renders/` (drop `.png` duplicates where `.jpg` exists)
  - `portfolio/modesto videos/` → `modesto/videos/`
  - Same pattern for `livermore` and `Sacremento` (note misspelling — fix to `sacramento`)
- Source files stay under `portfolio/` and are **gitignored**. Add `portfolio/` to `.gitignore` if not already.

### 1.2 Build script: `scripts/build-portfolio-assets.ts`
Use `sharp` (already in node_modules — check) for images, native `ffmpeg` (system) for video.

**Image pipeline per source file:**
- Read source
- Output 3 widths × 3 formats: `{slug}-{width}.{avif,webp,jpg}` at widths `[2400, 1600, 800]`
- Quality: AVIF 60, WebP 78, JPG 82. Strip metadata.
- Renders get all 3 widths; before/construction get `[1600, 800]` only (less critical).
- Compute width/height of source for each image; record into manifest.

**Video pipeline per Modesto source mp4:**
- Full walkthrough: H.264 baseline 1080p ~8Mbps + WebM VP9 1080p ~6Mbps + poster JPG at 2s mark
- One source designated as "hero loop": clip 6 seconds (use ffmpeg `-ss 0 -t 6`), strip audio, scale to 1280px wide, encode H.264 + WebM at low bitrate (~3MB target), poster

**ffmpeg commands (templates):**
```bash
# Full walkthrough H.264
ffmpeg -i "src.mp4" -c:v libx264 -preset slow -crf 23 -vf scale=1920:-2 -movflags +faststart -an out.mp4
# Full walkthrough VP9 WebM
ffmpeg -i "src.mp4" -c:v libvpx-vp9 -crf 32 -b:v 0 -vf scale=1920:-2 -an out.webm
# Hero loop (6s, no audio, 1280w)
ffmpeg -ss 0 -t 6 -i "src.mp4" -c:v libx264 -preset slow -crf 26 -vf scale=1280:-2 -an -movflags +faststart hero-loop.mp4
# Poster
ffmpeg -ss 2 -i "src.mp4" -frames:v 1 -q:v 3 poster.jpg
```

### 1.3 Generate manifest
Write `app/portfolio/assets.ts` with the typed structure above. Do NOT hand-edit — regeneration must be idempotent. Include alt text generated from filename pattern (e.g. "Chocolate Fish Modesto — render 1").

### 1.4 Validation
- Print final byte count per project. Modesto must be **<35MB total** including all videos.
- Reject the build if any video exceeds 15MB or any 2400w render exceeds 400KB.
- Update `next.config.mjs` if any new domains needed (none expected — all local).

## Voice & tone for editorial copy

- **Confident, terse, image-led.** Short sentences. Active voice.
- **Architectural / industrial design vocabulary.** "Pre-visualized." "Specified." "Built from a render."
- **Avoid hype words** (revolutionary, stunning, amazing, game-changing). Let the imagery be stunning; the copy stays understated.
- **Lean on the repeat-client signal.** "Trusted across three California locations." "Same brand, different build."
- **Sentences under ~22 words.** Punchy.

Example (Modesto description, ~3 sentences):

> Chocolate Fish's Modesto location was visualized end-to-end before a single fixture was ordered. We delivered photoreal renders, a walkthrough film, and tracked the build from foundation to opening day. The renders set the spec; the photographs prove they were hit.
